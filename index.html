    HTML

    • <main> </main> - główna część strony, tu wrzucamy zawartość strony
    • <header> </header> - nagłówek, gdy jest poza body powtarza się, na każdej stronie
    • <h1-h6> </h1-h6> - nagłówki
    • <footer> </footer> - stopka
    • <section> </section> - sekcja, wydziela spójną tematycznie część. Dzieli na sekcje. Przykład: sekcja sport w gazecie. W sekcji sport mogą być artykuły o piłce nożnej, siatkówce, itp.
    • <article> </article> - artykuł, niesie treść, treść tekstowa (ale i obraz, tabela), która może być zaprezentowana osobno
    • <p> </p> - paragraf, akapit

    CSS

    • budowa reguły CSS: selektor {wartość: wartość;} Selektor - wskazuje to co ma być ostylowane.
      W sekcji head w kodzie html musimy wpisać znacznik link: <link href="style.css" rel="stylesheet" type="text/css"
      Przykładowe znaczniki: {background / color: kolorbody / background / section / h1 / p / itp
      • Przykładowy zapis kolorów: {background / color: wybrana nazwa koloru; lub zapis szesnastkowy: #ffffff

    TERMINAL

    • cmd - uruchomienie terminala
    • c:/users - przejście do właściwego katalogu
    • cd c:/ - przejście do katalogu c
    • cd + pierwsza litera + TAB - przyspiesza wyszukanie pliku
    • dir - wyświetla zawartość katalogu
    • mkdir + nazwa katalogu - tworzenie katalogu
    • cd - wyświetlenie aktualnej ścieżki
    • cd nazwa katalogu - wejście do katalogu
    • cd .. - wejście do katalogu wyżej
    • cd ../.. - wejście 2 poziomy wyżej
    • code . - uruchomienie Visual Studio Code
    • code +nazwa pliku - otwarcie pliku

    GIT

      Komendy wpiywane w terminalu:

    • git --version - sprawdzenie wersji
    • git config --global user.name "imię i nazwisko"
    • git config --get user.name - sprawdzenie
    • git config --global user.email "adres e-mail"
    • git config --get user.email - sprawdzenie
    • git init + enter - zakładamy repozytorium - wchodzimy w terminalu do katalogu docelowego i wpisujemy ww komendę
    • git status - wyświetla status
    • git add . - dodawanie wszystkich plików z katalogu
    • git commit -m "tytuł zmiany" - zapisanie zmian, kommitowanie, angielska nazwa zmiany!
    • gitk - polecenie otwierające dokument/podgląd
    • git push - wypchnięcie zmian do git huba
    • clear - polecenie czyszczące okno
    • git lens - rozszerzenie pokazujące więcej szczegółów
    • Commit - stan projektu, czyli zapis kolejnych zmian
    • Kolejność działań: zmiany w kodzie w VSCode, następnie w terminalu: git status (na czerwono gdy są zmiany) / git add . /git commit -m "nazwa zmiany" / git push /
    • * strzałki góra/dół - wyświetlają poprzednio wpisywane komendy, co może przyspieszyć pracę
    • GIT HUB - programistyczna wizytówka, miejsce przetrzymywania repozytoriów

    DO ZAPAMIĘTANIA

    • camelCase: mySuperBoldHeader
    • kebab-case: my-super-bold-header
    • snake_case: my_super_bold_header
    • PascalCase: MySuperBoldHeader
    • Gdy stylujemy to nazwy angielskie zapisywane jako kebab-case (np.news-listing). Ważne - zapisuj nazwy zgodnie z ich funkcją. Element powinien mieć jedną klasę
    • Nie używać jednego ID do wielu elementów!
